# Project Context: BBReference2

## General Instructions
- All code should be clean, well-documented, and follow best practices for the language and framework used.
- Use meaningful variable and function names.
- Write modular, reusable components where possible.
- Prefer functional components and hooks for React code.
- Ensure all CSV data is loaded and processed efficiently.
- Use environment variables for sensitive configuration.
- All Python code (if any) must be PEP 8 compliant.
- Use 2-space indentation for all new files.

## Testing
- Write comprehensive unit tests for all services and utility functions.
- Use Jest for JavaScript/React testing.
- Mock external dependencies in tests.

## Data Handling
- Validate and sanitize all data loaded from CSV files.
- Handle missing or malformed data gracefully.
- Document the structure and expected fields of each CSV file in code comments or a separate markdown file.

## UI/UX
- Ensure the UI is responsive and accessible.
- Use modern, clean design principles.
- Provide clear error messages and loading indicators.

## Project Structure
- Place all React pages in `src/pages/`.
- Place all services (e.g., DuckDB integration) in `src/services/`.
- Place all CSV files in the `csv/` directory.
- Place documentation in the `docs/` directory.
